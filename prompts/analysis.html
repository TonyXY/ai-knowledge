分析类模板 🟡 进阶 ​

以下三个分析场景的 Prompt 模板，帮你在商业决策、产品规划中快速获取 AI 的分析支持。

模板一：数据分析解读 ​

场景说明：给 AI 提供原始数据或数据描述，让它给出有洞察的解读。

markdown
# 角色
你是一位资深数据分析师，擅长从数据中提取 actionable insights。

# 任务
分析以下数据，给出解读和行动建议。

# 数据
[粘贴表格、CSV、图表描述或文字数据]

# 分析维度
1. 整体趋势：数据呈现什么趋势？关键拐点在哪？
2. 异常识别：有没有异常值或需要关注的数据点？
3. 对比分析：对不同维度/时间段的数据进行对比
4. 归因分析：什么因素可能导致这些变化？

# 输出格式
- **关键发现**（3 条，每条 1-2 句话）
- **趋势解读**（一段话）
- **行动建议**（3 条可执行的建议，按优先级排序）
- **需要进一步关注的指标**

# 约束
- 不要过度解读——只从数据出发，不做无依据的推测
- 如果有数据不足以支撑结论的情况，明确指出

使用技巧：

  • 数据结构化程度越高，AI 的分析越精准
  • 可以补充说明行业背景，帮助 AI 更好地解读
  • 对不确定性较高的结论，追加要求："每条结论附上置信度（高/中/低）"

模板二：竞品分析 ​

场景说明：对市场竞品进行系统性分析，识别机会与威胁。

markdown
# 角色
你是一位产品战略分析师，擅长竞品分析。

# 任务
对以下竞品进行系统性分析。

# 竞品信息
- 竞品名称：[产品名]
- 目标用户：[用户画像]
- 核心功能：[列出 3-5 个核心功能]
- 定价策略：[价格区间或策略]
- 已知特点：[你知道的优缺点]

# 分析框架
请从以下维度分析：
1. **产品能力**：功能完整度、用户体验、差异化特点
2. **市场定位**：目标用户群、价格带、品牌形象
3. **优势分析**：竞品做得好的地方（我们该学什么）
4. **劣势分析**：竞品的短板和用户痛点（我们的机会在哪）
5. **威胁评估**：对我们产品的直接/间接威胁程度

# 输出格式
每个维度用 2-4 条 bullet points，最后附一段 100 字以内的总结。

# 约束
- 不评价"谁好谁坏"，只做客观分析
- 如果信息不足，标注"需进一步调研"

使用技巧：

  • 竞品信息越详细，分析越有价值
  • 可以粘贴竞品官网文字、用户评价等作为补充材料
  • 如果需要，追加要求："给出我们产品的应对策略建议"

模板三：SWOT 分析 ​

场景说明：对某个产品、项目或决策进行 SWOT 分析，辅助战略规划。

markdown
# 角色
你是一位商业策略顾问，擅长 SWOT 分析。

# 任务
对 [分析对象] 进行 SWOT 分析。

# 背景信息
- 分析对象：[产品 / 项目 / 公司 / 个人能力]
- 所处阶段：[起步期 / 成长期 / 成熟期 / 转型期]
- 核心信息：
  - 资源/优势：[你有什么]
  - 不足/短板：[你缺什么]
  - 市场环境：[机会在哪]
  - 竞争威胁：[风险在哪]

# 分析要求

## S — 优势（Strengths）
- 内部的、可控的优势
- 区别于竞争对手的能力

## W — 劣势（Weaknesses）
- 内部的、可控的不足
- 与竞争对手相比的短板

## O — 机会（Opportunities）
- 外部的、不可控的有利因素
- 市场趋势、政策红利、技术突破

## T — 威胁（Threats）
- 外部的、不可控的不利因素
- 竞争加剧、政策变化、替代品出现

# 输出格式
每个维度列出 3-5 条，每条 1-2 句话。最后给出一段 **SO/WT 策略建议**：
- SO（利用优势抓住机会）
- WT（弥补劣势规避威胁）

# 约束
- 基于提供的信息做分析，不虚构事实
- 信息不足的维度标注"待补充"

使用技巧：

  • SWOT 分析的质量取决于你提供的信息质量，背景信息越充分越好
  • 可以要求 AI 给每个条目标注重要程度（高/中/低）
  • 如果需要更深入，追加要求："基于 SWOT 结果，给出 3 个可执行的战略选项"

分析类 Prompt 通用原则 ​

原则说明
提供数据，不要光提问"根据以下数据……"比"你觉得……"更可靠
指定分析框架告诉 AI 用什么维度分析，而不是让它自由发挥
要求可执行的建议"给出 action items" 而不仅仅是"描述现象"
区分事实与推测要求 AI 标注哪些是数据支持的、哪些是合理推测

进阶阅读 ​

  • 结构化提示词 — 把分析需求组织成更清晰的结构
  • System Prompt 设计 — 固化分析风格和框架
