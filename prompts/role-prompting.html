角色设定 🟢 入门 ​

角色设定（Role Prompting） 是最简单却最高效的 Prompt 技巧之一。通过在 Prompt 开头给 AI 指定一个角色，你可以显著提升它在特定领域的输出质量。

为什么角色设定有效？ ​

当你让 AI "扮演"一个专家时，它会在内部激活与该角色相关的知识和表达模式。一个"资深 Python 工程师"的代码建议，远比一个无身份的通用回答更专业。

这背后的原理与 LLM 的训练方式有关——模型在海量数据中学会了不同角色的语言风格和知识结构。

基本语法 ​

角色设定的写法非常简单：

text
你是一位 [角色描述]。

放在 Prompt 的开头，效果最好。

角色设定示例 ​

技术专家 ​

text
你是一位有 10 年经验的资深 Python 后端工程师，精通 FastAPI、SQLAlchemy 和 Redis。
请帮我审查以下代码的性能问题，并给出优化建议。

教师 / 教育者 ​

text
你是一名经验丰富的小学三年级语文老师，擅长用故事和比喻让孩子理解复杂概念。
请用 8 岁孩子能听懂的语言，解释什么是人工智能。

编辑 / 审校 ​

text
你是一位专业的中文文字编辑，拥有 15 年出版行业经验。
请对以下文章进行校对，重点关注：
1. 错别字和语法错误
2. 表达不够流畅的句子
3. 段落结构是否合理
输出格式：先列出问题，再给出修改建议。

创意角色 ​

text
你是一位幽默的自媒体博主，风格类似脱口秀演员。
请用轻松吐槽的方式，介绍程序员一天的工作日常。
字数 200 字以内，不能出现脏话或人身攻击。

角色设定的进阶用法 ​

组合角色 ​

text
你同时是一位资深产品经理和有 UI/UX 设计背景的专家。
请从产品和设计两个角度分析这个功能方案。

角色 + 约束 ​

text
你是一位严格的代码评审员（Code Reviewer）。你的评审风格：
- 只关注代码质量和安全问题，不评价命名风格
- 每个问题必须给出修改建议的代码片段
- 如果代码没有明显问题，只说"LGTM 👍"

常见误区 ​

误区说明
角色过于模糊"你是一个专家" → 应具体到领域（"你是一位 Kubernetes 运维专家"）
角色过于宽泛"你什么都会" → 角色没有聚焦，效果等于没设定
角色放在末尾角色应放在开头，让模型尽早切换"人设"

进阶阅读 ​

  • System Prompt 设计 — 把角色固化到系统级指令中
  • 清晰指令五原则 — 角色设定 + 清晰指令 = 最佳效果
