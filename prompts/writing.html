写作类模板 🟡 进阶 ​

以下三个写作场景的 Prompt 模板，直接复制、填入你的信息即可使用。

模板一：小红书文案生成 ​

场景说明：为产品、活动或体验写一篇小红书风格的种草文案。

markdown
# 角色
你是一位小红书内容运营专家，擅长撰写高互动率的种草文案。

# 任务
为一款 [产品名称] 撰写一篇小红书笔记。

# 产品信息
- 产品名：[填入产品名称]
- 核心卖点：[填入 2-3 个核心卖点]
- 价格区间：[填入价格]
- 使用场景：[填入使用场景]

# 写作要求
1. 标题 15 字以内，emoji 开头，制造好奇心
2. 正文 200-300 字，分 3-5 个自然段
3. 包含 1-2 个真实使用场景描写
4. 语气亲切口语化，像朋友在聊天
5. 结尾加互动引导

# 输出格式
标题 + 正文 + 3 个话题标签

使用技巧：

  • 产品信息越详细，AI 生成的文案越有针对性
  • 可以在"写作要求"中指定 emoji 风格或禁用词
  • 如果结果不够口语化，追加一句："重写，语气要更像小姐妹聊天"

模板二：商务邮件撰写 ​

场景说明：写一封专业得体的商务邮件，适用于客户沟通、项目汇报、合作邀约等。

markdown
# 角色
你是一位专业的商务沟通顾问。

# 任务
撰写一封商务邮件。

# 邮件要素
- 收件人身份：[如：客户、上级、合作伙伴]
- 邮件目的：[如：汇报进度、请求延期、合作邀约]
- 核心信息：[需要传达的关键信息]
- 期望行动：[希望收件人做什么]

# 写作要求
1. 主题行简洁明确，不超过 20 字
2. 开头礼貌问候，开门见山说明来意
3. 中间段落逻辑清晰，用 bullet points 列出要点
4. 结尾明确下一步行动（Call to Action）
5. 语气 [正式 / 半正式 / 友好]

# 约束
- 不使用过于生硬的敬语（如"此致 敬礼"）
- 不超过 200 字

使用技巧：

  • 正式程度根据收件人调整：客户用"正式"，同事用"半正式"
  • 如果是回复邮件，在"核心信息"中附上对方邮件的要点
  • 可以追加要求："在邮件末尾加一句表示感谢的话"

模板三：工作报告总结 ​

场景说明：将零散的工作内容整理成结构化的周报、月报或项目总结。

markdown
# 角色
你是一位擅长结构化表达的工作汇报专家。

# 任务
根据以下零散信息，整理成一份工作报告。

# 原始信息
[粘贴你的工作日志、聊天记录、会议笔记等]

# 报告结构
1. **本周完成** — 已完成的关键任务（3-5 条）
2. **进行中** — 正在推进的工作及进度
3. **遇到的问题** — 遇到的阻塞点和待解决问题
4. **下周计划** — 下周的重点工作（3-5 条）
5. **需要的支持** — 需要上级或同事协助的事项

# 写作要求
- 每条用 1-2 句话，简洁有力
- 使用动词开头的短句（"完成了 XX"、"推进了 XX"）
- 用数字量化成果（"提升 20%" > "有所提升"）
- 语言务实，不夸大、不画饼

使用技巧：

  • 原始信息可以非常口语化，AI 会帮你提炼
  • 如果信息不够，AI 会基于已有内容合理补全，但需人工核实
  • 可以在"约束"中追加："如果某项缺少信息，标注为'N/A'而非编造"

通用写作技巧 ​

技巧说明
先用模板，再微调第一次用默认模板生成，再根据结果微调约束条件
提供范文在模板中加入"风格参考"段落，附上你喜欢的范文
迭代优化对不满意的部分，单独追加指令："把第二段改得更活泼一些"
保存好用的模板把调试好的 Prompt 保存为个人模板，下次直接复用

进阶阅读 ​

  • Few-Shot / Zero-Shot — 给模板加示例，效果更佳
  • 结构化提示词 — 理解结构化 Prompt 的设计思路
