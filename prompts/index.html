💬 提示词工程 ​

提示词（Prompt）是你与 AI 沟通的桥梁。一个好的提示词，能让 AI 从"答非所问"变成"精准输出"。

本模块按三个层次组织，建议从基础篇开始，再到进阶技巧，最后用场景模板实战练习。

🟢 入门 无需前置知识 · 🟡 进阶 建议先掌握基础篇

🟢 基础篇 ​

从这里入门，建立对提示词的基本认知。

什么是 Prompt 🟢 入门

理解 Prompt 的本质——你给 AI 的指令。介绍 System / User / Assistant 三部分结构，以及好 Prompt 的标准。

清晰指令五原则 🟢 入门

写出好 Prompt 的五条核心原则：明确任务、提供上下文、指定格式、给出示例、约束边界。每条原则配对比示例。

角色设定 🟢 入门

让模型扮演特定角色（专家、教师、编辑等），显著提升特定领域的输出质量。

Few-Shot / Zero-Shot 🟢 入门

Zero-Shot 不给示例直接问，Few-Shot 先给 2-3 个示例再提问。理解两者的区别和适用场景。

🟡 进阶技巧 ​

掌握这些技巧，你的提示词水平将超越大多数使用者。

System Prompt 设计 🟡 进阶

System Prompt 是预设的系统级指令，影响模型全程行为。好的设计 = 角色 + 规则 + 输出约束 + 边界条件。

结构化提示词 🟡 进阶

用 Markdown / XML / JSON 格式组织 Prompt，让 AI 更精确地理解你的意图。比纯文本结构更清晰。

🟡 场景模板 ​

实战中最常用的三类 Prompt 模板，可直接复制修改后使用。

写作类模板 🟡 进阶

小红书文案、商务邮件、工作报告——三个高频写作场景的 Prompt 模板和技巧。

编程类模板 🟡 进阶

代码生成、Debug 排查、代码重构——三个编程场景的 Prompt 模板和期望效果。

分析类模板 🟡 进阶

数据分析解读、竞品分析、SWOT 分析——三个分析场景的 Prompt 模板。
