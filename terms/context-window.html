🟢 入门

上下文窗口 ​

上下文窗口（Context Window）决定了 AI 模型在一次对话中能同时"看到"多少信息。它是衡量模型实际可用能力的关键指标之一。

一句话解释 ​

上下文窗口是 LLM 单次处理的最大 Token 数量——包括你输入的所有内容、历史对话、以及模型即将输出的内容。超过这个限制，模型就会"忘记"最早的信息。

通俗类比 ​

想象你在一张桌子上办公：

  • 上下文窗口 = 桌子的大小
  • 你的对话历史 = 桌上的文件
  • 新问题 = 你要处理的新文件

桌子越大（上下文窗口越大），你能同时铺开的文件就越多（对话历史能保留更多）。一旦文件多到放不下，你必须把最早的文件收起来（模型忘记早期信息）。

不同模型桌子大小差异极大：

模型上下文窗口
GPT-3.5（初版）4K Token（一张小课桌）
GPT-48K~128K
Claude 3.5 Sonnet200K（一整张会议桌）
Gemini 1.5 Pro1M+（一整间教室的地板）

技术定义 ​

上下文窗口的大小由模型架构（特别是位置编码方式）决定。在 Transformer 中，自注意力机制的计算复杂度随序列长度平方增长（O(n²)），因此扩大上下文窗口需要大量工程优化。

实际应用中需要注意：并不是上下文窗口越大就越好。更大的窗口意味着更慢的响应速度、更高的 API 成本，而且模型在超长上下文末尾的"注意力"往往会衰减。

关联术语 ​

  • Token —— 上下文窗口的计数单位
  • Prompt —— 每次对话消耗上下文窗口容量
