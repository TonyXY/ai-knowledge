# 参数 (Parameters)

当你看到 "GPT-4 有 1.7 万亿参数" "Llama 3 有 70B 参数" 这样的数字时，它们到底是什么意思？

## 一句话解释

参数（Parameters）是模型在训练过程中学到的一组数值——就好比人脑中神经元的连接强度。这些数值决定了输入数据经过模型后会产生什么输出。参数越多，模型通常越 "聪明"，但训练和运行成本也越高。

## 通俗类比

把模型想象成一个**超级多按钮的调音台**：

- 每个参数 = 调音台上的一个小旋钮
- 训练过程 = 一位专业调音师花几个月时间，反复播放各种音乐，每次微调所有旋钮，直到音效完美
- 推理过程 = 用调好的旋钮位置直接播放，不再调整

70 亿参数（7B）的模型，相当于有 **70 亿个旋钮**需要调——这就是为什么训练大模型需要天价计算资源。

## 技术定义

参数在神经网络中以**矩阵形式**存储，通常是浮点数（FP16、FP32 等）：

- **参数量级**：7B（70 亿）、70B（700 亿）、数百 B（数千亿）、万亿+
- **存储估算**：1 个参数如果用 FP16（2 字节），7B 模型大约占 **14GB 显存**（仅参数，不含推理开销）
- **算力需求**：参数量翻倍 ≈ 推理算力翻倍

参数数量 ≠ 一切。以下是影响模型能力的核心要素：

| 因素 | 说明 |
|------|------|
| 参数数量 | 模型的 "容量"，越高理论上能学更多 |
| 训练数据质量 | 垃圾数据训练不出好模型 |
| 训练数据量 | 参数多但数据少 = 过拟合 |
| 架构设计 | MoE 架构可让部分参数 "休眠"，降低推理成本 |
| 后训练优化 | RLHF、DPO 等对齐技术对实际体验影响巨大 |
