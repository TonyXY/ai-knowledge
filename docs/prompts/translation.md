# 翻译类模板 <DifficultyBadge level="beginner" />

翻译是 Prompt 最实用的场景之一。相比传统翻译工具，AI 翻译的优势在于可以指定语言风格、保留术语、理解上下文，甚至进行本地化适配。以下四个模板覆盖了从日常到专业的翻译需求。

---

## 模板一：通用翻译

**场景说明**：将一段文字翻译成指定语言，同时控制语气和风格。

```text
请将以下中文翻译成英文，要求：
- 风格：自然口语化，适合日常对话
- 保留原文的情感和语气
- 如果遇到中文特有的表达（如成语、俗语），请用意译而非直译

原文：
「这事儿说来话长，但简单讲就是——我那天出门没看黄历。」
```

**使用技巧**：
- 在"要求"中指定目标受众（如"面向美国青少年"），AI 会用更地道的表达
- 对于文学性文本，可以要求"保留原文的修辞手法"
- 不确定翻译质量时，追加一句：`请逐句对比原文和译文，说明你的翻译策略`

## 模板二：商务翻译

**场景说明**：翻译商务邮件、合同条款、产品介绍，要求正式语气和术语一致性。

```text
# 角色
你是一名资深商务翻译，精通中英双语，熟悉国际贸易术语。

# 任务
将以下中文商务邮件翻译成英文。

# 要求
1. 语气正式专业，符合国际商务惯例
2. 术语保持一致：如「报价单」统一译为 "quotation"，「交货期」统一译为 "delivery lead time"
3. 保留原文的礼貌用语和分寸感
4. 如果有模糊表达，选择更正式的理解方向

# 术语表（如有）
- 报价单 → quotation
- 交货期 → delivery lead time
- 付款条件 → payment terms
- 质保期 → warranty period

原文：
「感谢贵司的及时回复。关于本次报价，我们希望确认以下几点：
1. 交货期能否从 30 天缩短至 20 天；
2. 付款条件是否支持 30% 预付 + 70% 货到付款；
3. 质保期内的售后服务包含哪些内容。

如以上条款可以满足，请发送正式报价单至我司邮箱。」
```

**使用技巧**：
- 术语表越详细，AI 的翻译越一致——这对长文档翻译尤其重要
- 可以上传公司已有的翻译记忆库（TM）作为参考
- 翻译合同条款时，建议追加：`请用括号标注可能有歧义的表达，并给出备选翻译`

## 模板三：技术文档翻译

**场景说明**：翻译技术文档、API 说明、README 等，核心要求是代码和术语不译。

```text
# 任务
将以下技术文档从英文翻译成中文。

# 关键规则
1. **代码不译**：所有代码块、变量名、函数名、命令行保持原样
2. **术语统一**：
   - middleware → 中间件
   - hook → 钩子
   - state → 状态
   - dependency → 依赖
   - 首次出现的术语，在中文后加括号标注英文原文
3. 技术描述的准确性优先于语言流畅性
4. 保留原文的 Markdown 格式和代码块结构

原文：
<待翻译的技术文档>
```

**使用技巧**：
- 把这份 Prompt 保存为固定模板，每次粘贴待翻译内容即可
- 如果文档包含 API 参数说明，可以追加规则："参数名不译，参数说明翻译"
- 对于大型文档，分段落翻译比一次性翻译质量更高

## 模板四：本地化适配

**场景说明**：不仅仅是翻译，还要考虑目标市场的文化差异、习惯表达和本地规范。

```text
# 角色
你是一名资深本地化专家，擅长将产品内容适配到中国大陆市场。

# 任务
将以下英文 App 介绍页面进行本地化处理（不是字面翻译）。

# 本地化要求
1. 将英制单位转换为公制（如 mile → 公里，pound → 公斤）
2. 将西方节日、文化引用替换为中国用户熟悉的对应内容
3. 调整语气：英文幽默可能不适合中文市场，请用更真诚、务实的语气
4. 保留产品的核心卖点，但用中国用户关心的表达方式重述
5. 格式上：英文标题（供参考）→ 中文标题 → 中文正文

原文：
<待本地化的内容>
```

**使用技巧**：
- 本地化不是翻译，是「再创作」——给 AI 充分的修改自由度
- 可以要求 AI 先给出"本地化策略说明"，确认方向后再生成内容
- 对于敏感内容（政治、宗教、文化符号），建议追加要求：`如果遇到可能在中国市场引发争议的内容，请标注并给出修改建议`

## 翻译类 Prompt 通用原则

1. **告诉 AI 受众是谁**——给小学生看的翻译和给 CEO 看的翻译完全不同
2. **提供术语表**——这是翻译质量一致性的关键
3. **明确不可译内容**——代码、品牌名、专有名词
4. **分轮次校对**——第一轮翻译 → 第二轮润色 → 第三轮术语检查
5. **善用回译验证**——让 AI 把译文再翻回原文，对比差异
