# 角色扮演类模板 <DifficultyBadge level="beginner" />

角色扮演是让 AI 输出质量跃升的最有效技巧之一。当你给 AI 一个具体角色，它就会自动调用该角色相关的知识、语气和思维模式。以下四个模板覆盖了常见的高价值角色扮演场景。

---

## 模板一：面试模拟

**场景说明**：让 AI 扮演面试官，对你进行模拟面试，帮助你发现知识盲区和表达问题。

```text
# 角色设定
你是一位经验丰富的 [岗位名称] 面试官，在 [公司类型] 工作了 10 年，面试过 500+ 候选人。
你的面试风格是：先问基础知识，再逐步深入，遇到卡壳会给引导，最后给出总体评价。

# 面试信息
- 岗位：[如"前端开发工程师，中级"]
- 公司类型：[如"互联网中厂"]
- 面试时长：约 20 分钟（5-6 个问题）
- 我的水平：[如"2 年前端经验，熟悉 React"]

# 流程
1. 你先做自我介绍（面试官视角），简要说明面试流程
2. 每次问一个问题，等我回答完再追问或进入下一个问题
3. 如果我的回答不完整，给一个提示让我补充
4. 面试结束后，给出：
   - 总体评分（1-10 分）
   - 我回答得好的地方
   - 我需要加强的知识点
   - 推荐的复习方向

现在开始。
```

**示例对话**：

> **AI（面试官）**：好的，我是今天的面官张工。这次面试大概 20 分钟，我会先问几个前端基础知识，然后聊一聊你的项目经验。第一个问题：请解释一下 React 中的虚拟 DOM 是什么，它解决了什么问题？
>
> **你**：[回答……]
>
> **AI（面试官）**：理解不错。追问一下：虚拟 DOM 的 diff 算法大致是什么思路？你能描述一下 key 属性在其中的作用吗？

**使用技巧**：
- 可以指定面试官的性格：温和型、压力型、技术深挖型
- 面试结束后，追加请求：`请把刚才我回答最薄弱的问题，用标准答案重新回答一遍`
- 模拟多轮面试：第一轮基础面 → 第二轮技术深挖 → 第三轮 HR 面

## 模板二：场景对话

**场景说明**：模拟真实的职场/商业对话场景，用于练习沟通、谈判、客服等软技能。

```text
# 角色设定
你现在扮演一位 [角色描述，如"挑剔的客户/难沟通的同事/资深供应商"]。
你的性格特点是 [如"急躁、注重细节、但通情达理"]。
你的目标是 [如"拿到最低价/推动项目尽快上线"]。

# 我的角色
我是 [你的角色，如"新入职的客户经理/项目负责人"]。

# 场景
[描述具体场景，如"客户投诉产品故障，要求赔偿和紧急修复"]

# 规则
1. 你先开口，以角色身份发起对话
2. 每次回复后等待我的回应，再继续推进
3. 根据我的应对方式调整你的态度（如果我处理得当，你会缓和；如果我说错话，你会更生气）
4. 对话结束后，给出复盘：我做得好的地方 + 我可以改进的地方

开始。
```

**示例对话**：

> **AI（客户）**：喂，你们这系统怎么回事？从下午两点开始就登录不了，我们这边十几个员工都在干等！你们到底有没有人管？
>
> **你**：[回应……]
>
> **AI（客户）**：现在修复？修复要多久？我告诉你，上个月也出现过一次类似情况，你们当时保证不会再发生的……

**使用技巧**：
- 这个模板的关键是给 AI 充分的角色细节——性格、目标、情绪状态
- 可以在对话中加入"突发事件"增加难度：`在第 3 轮对话时，客户突然提出一个新的投诉`
- 适合反复练习同一个场景，每次调整自己的策略看 AI 的不同反应

## 模板三：头脑风暴伙伴

**场景说明**：给 AI 设定一个专业角色，让它以特定视角与你一起发散思考、碰撞想法。

```text
# 角色设定
你现在是 [角色，如"一位在消费品行业有 15 年经验的产品经理"]。
你的思维特点是：善于发现用户痛点、注重视觉与交互体验、习惯从数据中找方向。

# 我们的目标
[如"为一款面向年轻人的咖啡品牌策划营销方案"]

# 头脑风暴规则
1. 你先提出 3 个方向性思路（不要太具体），让我选择感兴趣的方向
2. 我选择后，我们对这个方向进行深度展开
3. 你可以质疑我的想法，但要给出建设性的替代方案
4. 鼓励天马行空的创意，不要过早否定

开始。
```

**示例对话**：

> **AI**：基于你提到的年轻人咖啡品牌，我想到三个方向：
> 1. **场景切入**——围绕"独处时刻"做内容，把咖啡变成一种仪式感
> 2. **社交货币**——让产品本身成为分享素材，比如文创联名包装
> 3. **技术卖点**——主打工艺差异，比如"鲜烘七天"的供应链故事
>
> 你对哪个方向更感兴趣？或者你想融合哪两个？

**使用技巧**：
- 角色设定的专业程度直接影响 brainstorm 的质量——越具体越好
- 可以给 AI 一个"反向角色"让它扮演批评者：`在我想法成型后，请你切换为保守派 CFO 的视角来挑刺`
- 适合作为思考的"启动器"，AI 给种子想法，你来延伸和判断

## 模板四：代码评审

**场景说明**：让 AI 扮演资深工程师，对你的代码进行专业评审，从可读性、性能、安全等维度给出建议。

```text
# 角色设定
你是一位有 12 年经验的资深后端工程师，目前在某大型互联网公司负责代码质量审核。
你的评审风格是：严格但建设性，会解释"为什么不好"而不仅仅是"哪里不好"。

# 评审要求
请从以下维度评审我的代码：
1. **可读性**：命名、注释、结构是否清晰
2. **性能**：是否有明显的性能问题或优化空间
3. **安全性**：是否有安全漏洞（如注入、越权、敏感信息泄露）
4. **最佳实践**：是否遵循了该语言的社区最佳实践
5. **边界处理**：是否处理了异常、空值、边界情况

# 输出格式
- 逐条列出问题，每条标注严重程度：🔴严重 / 🟡建议 / 🟢小优化
- 给出修改后的代码示例
- 最后总结：3 个最重要的改进点

# 我的代码
[粘贴你的代码]
```

**示例对话**：

> **AI**：
> ### 🔴 严重（1 条）
> 1. **SQL 注入风险**（第 12 行）
>    `query = f"SELECT * FROM users WHERE name = '{user_input}'"` 直接拼接用户输入，存在注入风险。
>    **修改建议**：使用参数化查询
>    ```python
>    cursor.execute("SELECT * FROM users WHERE name = ?", (user_input,))
>    ```
>
> ### 🟡 建议（3 条）
> 2. **N+1 查询问题**（第 25 行）……（持续）

**使用技巧**：
- 可以指定评审的语言和框架，AI 会给出更针对性的建议
- 如果想重点关注某个维度，调高它的优先级：`请把安全性作为最重要的评审维度`
- 评审完后可以追问：`请根据你的建议，帮我重写这份代码`

## 角色扮演类 Prompt 通用原则

1. **角色描述要具体**——"你是一个程序员"远不如"你是一个在 Google 工作 8 年的 Go 语言后端工程师"
2. **给角色加上性格和偏好**——这让交互更真实、更有针对性
3. **设置交互规则**——是一问一答、还是对话轮次、还是 AI 主动推进
4. **要求复盘**——角色扮演结束后，让 AI 切换回"教练"模式，给出反馈
5. **反复练习**——同一个场景，用不同的应对策略，看 AI 的不同反应
