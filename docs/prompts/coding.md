# 编程类模板 <DifficultyBadge level="intermediate" />

以下三个编程场景的 Prompt 模板，让你的 AI 编程效率翻倍。

---

## 模板一：代码生成

**场景说明**：让 AI 根据需求描述生成代码，适用于快速原型、工具函数、配置脚本等。

```markdown
# 角色
你是一位精通 [语言/框架] 的高级工程师。

# 任务
实现一个满足以下需求的 [函数/组件/模块]。

# 需求描述
[用自然语言描述你要实现的功能]

# 技术约束
- 语言：[Python / TypeScript / Go / etc.]
- 框架：[FastAPI / React / Gin / etc.]
- 依赖限制：[只能用标准库 / 可用 numpy / etc.]

# 代码要求
1. 包含完整的类型注解（如有类型系统）
2. 关键逻辑加注释说明
3. 处理边界情况（空输入、异常值等）
4. 优先可读性，而非极致简洁

# 输出格式
- 先简要说明实现思路（2-3 行）
- 然后给出完整代码
```

**期望效果**：AI 会输出可运行的代码 + 实现思路，减少你后期的调试时间。

**使用技巧**：
- 需求描述越具体，代码越贴近你的预期
- 可以要求 AI 同时写出单元测试
- 如果代码有问题，把错误信息贴回去让它修复

## 模板二：Debug 排查

**场景说明**：遇到 Bug 时，把错误信息和相关代码给 AI，快速定位根因。

```markdown
# 角色
你是一位精通 [语言/框架] 的调试专家。

# 问题描述
[简要描述：你预期什么行为，实际发生了什么]

# 错误信息
```text
[粘贴完整的错误堆栈或日志]
```

# 相关代码
```[语言]
[粘贴出问题的代码片段，尽可能提供上下文]
```

# 环境信息（可选）
- 语言版本：[如 Python 3.11]
- 框架版本：[如 React 18.2]
- 操作系统：[如 macOS 14]

# 输出要求
1. **根因分析**：错误的最可能原因（1-2 句话）
2. **修复方案**：给出修改后的代码
3. **预防建议**：如何避免类似问题
4. 如果信息不足无法判断，说明还需要什么信息
```

**期望效果**：AI 能快速定位 80% 的常见 Bug，并给出可用的修复代码。

**使用技巧**：
- **错误信息一定要完整**，不要只贴最后一行
- 提供触发 Bug 的输入数据
- 如果修复后仍有问题，可以继续追问

## 模板三：代码重构

**场景说明**：让 AI 审查并改进已有代码的可读性、性能或架构。

```markdown
# 角色
你是一位专注于代码质量的资深工程师。

# 任务
审查并重构以下代码。

# 重构目标
- [ ] 提高可读性和可维护性
- [ ] 优化性能瓶颈
- [ ] 消除重复代码（DRY 原则）
- [ ] 改进错误处理

# 代码
```[语言]
[粘贴待重构的代码]
```

# 约束
- 不改变外部接口和返回值格式
- 保持现有测试通过
- 重构后的代码必须有类型注解

# 输出格式
1. **当前问题**：列表形式，每条说明问题和位置
2. **重构后代码**：完整代码
3. **改进说明**：解释关键改动点及其好处
```

**期望效果**：AI 会给出结构化的改进分析 + 可直接使用的新代码。

**使用技巧**：
- 如果只需要特定类型的重构（如只做性能优化），在"重构目标"中只勾选对应项
- 可以要求 AI 给出渐进式重构步骤（改一点、测试、再改一点）
- 大文件建议分模块逐步重构，每次只给 AI 一个模块

---

## 编程 Prompt 通用原则

| 原则 | 说明 |
|------|------|
| 代码要完整 | 让 AI 看到足够的上下文，不要只给孤立的几行 |
| 错误要详细 | 粘贴完整错误堆栈，不要只描述"报错了" |
| 反馈要具体 | "不用 reduce，用 for 循环重写" 好于 "改一下" |
| 分步处理 | 大任务拆成小步骤，逐步让 AI 实现 |
