# 用 Claude 分析文档

Claude 的 200K 上下文窗口是分析长文档的利器。本篇教你如何高效利用它来阅读论文、分析合同、处理技术文档。

## 准备工作

- 注册 [Claude](https://claude.ai) 账号（免费版即可，但 Pro 版体验更好）
- 准备一份要分析的文档（PDF、Word、TXT 等格式都支持）

---

## 第 1 步：上传文档

打开 Claude 网页版，点击输入框旁的 📎 附件图标，选择你要分析的文档。

Claude 会自动"阅读"整个文档。200K tokens 的上下文意味着它可以一次性处理：
- 约 15 万英文单词
- 约 200 页 PDF
- 整本中等篇幅的书

### 支持的文件格式

| 格式 | 支持 | 说明 |
|------|------|------|
| PDF | ✅ | 扫描版 PDF 也能处理（OCR） |
| Word (.docx) | ✅ | 保留格式 |
| Excel (.xlsx) | ✅ | 可以分析表格数据 |
| TXT | ✅ | 纯文本 |
| 图片 | ✅ | 支持图像理解 |
| PPT | ✅ | 可以分析演示文稿 |

---

## 第 2 步：提出分析要求

文档上传后，就可以开始提问。根据文档类型，提问方式不同。

### 场景 1：分析学术论文

```
你：请分析这篇论文，回答以下问题：
1. 论文的核心贡献是什么？
2. 作者提出了什么新方法？和现有方法有什么区别？
3. 实验结论是什么？有哪些关键数据？
4. 有什么局限性或未来工作方向？

用中文回答，结构要清晰。
```

Claude 会给出结构化的分析：

```
## 论文核心贡献
本文提出了 XXX 方法，解决了 XXX 问题...

## 方法论
创新点在于：
1. ...
2. ...

## 实验结论
在 XXX 数据集上，该方法比 baseline 提升了 15%...

## 局限性
作者指出...
```

### 场景 2：审查合同

```
你：请审查这份合同，找出以下风险点：
1. 违约责任条款是否对某一方不公平？
2. 知识产权归属有没有隐患？
3. 解除条款的条件是否合理？
4. 是否有隐藏的自动续约条款？

请逐条列出风险，用"⚠️ 高风险 / ⚡ 中风险 / ℹ️ 低风险"标注等级。
```

### 场景 3：分析技术文档

```
你：这是我们的项目技术文档。请帮我：
1. 总结系统架构
2. 画出核心流程（用文字描述即可）
3. 找出文档中矛盾或不清楚的地方
4. 列出所有 API 接口及其功能
```

### 场景 4：对比多个文档

```
你：我上传了两份简历。请用表格对比两人的背景：
- 教育背景
- 工作经历
- 技能匹配度
- 优缺点

最后给出你的推荐（假设招聘岗位是前端开发工程师）。
```

## 第 3 步：利用长上下文深入分析

Claude 的强项是一次性处理大量信息。充分利用这一点：

### 逐章分析

```
你：这本书一共 12 章，我已经全部上传。请：
1. 给出每章的一句话摘要
2. 指出各章之间的逻辑关系
3. 提炼全书的 3 个核心观点
```

### 跨文档关联

```
你：我上传了同一项目的需求文档、技术方案和测试报告。
请分析：技术方案是否完整覆盖了需求文档中的所有需求点？
哪些需求没有对应的技术方案？
```

### 追问细节

```
你：你刚才提到的 XXX 方法，论文里有没有解释为什么选这个参数值？
```

Claude 会回到文档中查找细节——全程记住对话历史。

---

## 第 4 步：获得结构化输出

对于需要复用的分析结果，可以让 Claude 以特定格式输出。

### 要求 Markdown 表格

```
你：请把这篇文章中提到的所有技术方案整理成表格：
| 方案名称 | 适用场景 | 优点 | 缺点 | 复杂度 |
```

### 要求 JSON

```
你：请把合同中的关键条款提取为 JSON 格式：
{
  "parties": ["甲方", "乙方"],
  "effective_date": "2026-01-01",
  "payment_terms": { ... },
  "termination": { ... }
}
```

## 实用 Prompt 模板

### 论文精读

```
请作为我的研究助手，精读这篇论文：
1. 用一段话概括核心思想（限制 200 字）
2. 列出 3 个最重要的创新点
3. 解释 Figure 3 和 Figure 5 说明了什么
4. 这个方法在我描述的以下场景中能用吗：[描述你的场景]
5. 推荐 3 篇相关论文（不要求真的找到，给出方向即可）
```

### 会议纪要

```
请把这 2 小时的会议录音转写内容整理成纪要：
1. 结论性决定（标出责任人）
2. 待办事项（TODO list）
3. 分歧点（未达成共识的事项）
4. 下次会议建议议程

格式：Markdown，每一项前面用 checkbox。
```

## 总结

| 步骤 | 操作 |
|------|------|
| 1 | 上传文档（PDF / Word / TXT 等） |
| 2 | 提出结构化分析要求 |
| 3 | 利用长上下文深入提问 |
| 4 | 获得结构化输出（Markdown / JSON） |

Claude 的 200K 上下文窗口让它成为长文档分析的首选工具。搭配清晰的 Prompt，你可以快速从海量信息中提炼出真正有价值的内容。
