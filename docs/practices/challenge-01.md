# 挑战：用 AI 搭建一个个人网站

> ⚠️ 这是一个**开放式挑战任务**，不是教程。没有标准答案，不提供分步指导。你需要自己想办法完成。

## 挑战说明

本篇不是教程——它没有"第 1 步、第 2 步"。你拿到的是一份**需求规格书**，你的任务是用 AI 工具独立完成它。

**规则很简单：**

- 你只能使用 AI 工具（ChatGPT、Claude、Cursor、Windsurf、Copilot 等）
- 没有标准答案——怎么实现、用什么技术栈，完全由你决定
- 最终交付物是一个**已部署上线**、可访问的个人简历网站
- 过程中遇到的任何问题，优先问 AI 而不是查文档

这个挑战的设计思路是：在实际工作中，你不会拿到"保姆级教程"，你拿到的是需求。能不能拆解需求、向 AI 提出正确的问题、把 AI 产出的代码整合成完整项目——这些才是真正需要练习的能力。

---

## 需求规格

你需要搭建一个**个人简历网站**，包含以下四个页面或区块：

### 1. 首页

- 个人简介段落（至少 50 字）
- 一个头像占位区域（用纯 CSS 实现，不需要真实照片）
- 一句**个人标语**（slogan），突出展示
- 页面整体有明确的视觉焦点，让访客一眼知道你是谁

### 2. 技能页

- 列出至少 6 项技能，分为前端、后端、设计等类别
- 每项技能有**可视化标识**——进度条、星级、标签皆可
- 类别之间在视觉上有清晰区分
- 整体布局在移动端不塌陷

### 3. 项目页

- 展示至少 **3 个完整项目**
- 每个项目包含：项目名称、一句话描述、一个截图占位区域、技术栈标签
- 截图占位用纯 CSS 实现（灰色矩形 + 图标即可）
- 项目之间通过卡片形式区分，视觉层次分明

### 4. 联系方式

以下二选一：
- **表单方案**：包含姓名、邮箱、留言三个字段，有提交反馈
- **社交链接方案**：至少 4 个平台链接（GitHub、邮箱、LinkedIn 等），带 hover 动效

额外加分：两种都做。

### 技术要求（硬性条件）

- **纯前端实现**：HTML + CSS + JavaScript，或使用 React/Vue 等框架（需能本地运行）
- **响应式设计**：手机（< 768px）、平板（768px - 1024px）、桌面（> 1024px）三种尺寸均正常显示
- **部署上线**：发布到 GitHub Pages 或 Vercel，确保可通过公开 URL 访问
- **使用 AI 工具完成全部编码**：不手动写核心逻辑，所有代码由 AI 生成、你负责整合与调试
- **不含外部图片**：所有视觉元素用 CSS 实现
- **不引入后端服务**：表单提交可以用 `mailto:` 或静默处理

## 可用工具

| 类别 | 工具 | 用途 |
|------|------|------|
| AI 编程 IDE | Cursor / Windsurf / Copilot | 在编辑器中直接生成和修改代码 |
| AI 对话 | ChatGPT / Claude | 生成代码片段、设计方案、Debug |
| 部署平台 | GitHub Pages / Vercel | 免费部署静态网站 |

建议组合：用 ChatGPT 或 Claude 先讨论整体设计和技术方案，然后用 Cursor 或 Windsurf 在编辑器中逐步实现。

## 评估标准

完成挑战后，从以下四个维度自评：

| 维度 | 说明 |
|------|------|
| **功能完整性** | 四个页面/区块全部实现，交互逻辑正常，没有明显 bug |
| **代码质量** | HTML 语义化、CSS 无大量冗余、JS 逻辑清晰、有适当注释 |
| **视觉设计** | 色彩搭配协调、排版整洁、四个页面风格统一、响应式断点合理 |
| **AI 利用效率** | 向 AI 提问的质量如何？是否需要反复修正才能得到可用的代码？ |

达到所有标准即为"挑战通过"。不必追求完美——追求"能用 + 好看"即可。

## 提示（非步骤）

以下不是操作步骤，而是策略建议——你可以采纳，也可以用自己的方式：

- **先设计方案，再写代码**：用 AI 生成一份完整的页面设计方案（布局草图、配色、字体选择），确认可行后再动手
- **分模块推进**：按页面逐个完成，每完成一个模块就让 AI Review 一次，趁早发现问题
- **先让 AI 试错**：遇到报错或效果不对，把现象和代码一起扔给 AI，让它先给出诊断——大多数问题 AI 都能自己解决
- **第一次生成的代码几乎都不完美**：把 AI 当结对编程伙伴，持续对话、迭代优化，三到五轮对话后的代码质量会明显提升
