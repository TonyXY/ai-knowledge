## 选择题

### Q1：RAG 的完整流程是什么？

    A. 只检索 → 不生成

    B. 检索 → 生成（先检索相关知识再让 LLM 回答）

    C. 只生成 → 不检索

    D. 随机生成 → 再检索确认

<details>
<summary>查看答案</summary>

**正确答案：B**

RAG（Retrieval-Augmented Generation，检索增强生成）的完整流程是**「先检索，再生成」**：

1. **索引阶段**（离线）：文档 → 分块 → Embedding（向量化）→ 存入向量数据库
2. **检索阶段**（运行时）：用户问题 → Embedding → 在向量数据库中检索 Top-K 相关片段
3. **生成阶段**：将问题 + 检索到的相关片段一起交给 LLM，让 LLM 基于这些"参考资料"生成回答

这种架构解决了 LLM 的两大痛点：
- **知识截止日期**：通过检索获取最新信息
- **幻觉问题**：有据可查，回答可追溯来源

</details>

### Q2：Streaming（流式响应）的主要作用是什么？

    A. 让回答更准确

    B. 减少首字延迟，提升用户体验

    C. 降低 API 费用

    D. 增加回答长度

<details>
<summary>查看答案</summary>

**正确答案：B**

Streaming 的核心目的是**减少首字延迟（Time to First Token, TTFT）**，从而提升用户体验：

- **非流式**：LLM 生成完所有 token 后一次性返回，用户需要等待整个回答完成才能看到内容（可能等几秒到几十秒）
- **流式**：LLM 每生成一个 token 就立即发送，用户可以**实时看到文字逐字出现**，就像真人在打字

虽然技术上不改变回答的准确性和最终长度，但这种"即时反馈"极大改善了交互体验——你不会盯着空白屏幕等答案，而是能同步看到 AI 的思考过程。这也是 ChatGPT、Claude 等所有主流产品都使用流式输出的原因。

</details>

### Q3：在用 AI 编程时，第一步应该做什么？

    A. 直接让 AI 写完整代码

    B. 先清晰描述需求和约束条件

    C. 让 AI 先选框架

    D. 先把所有代码写完再问 AI

<details>
<summary>查看答案</summary>

**正确答案：B**

**「先清晰描述需求和约束条件」**是 AI 编程的第一步，也是最关键的一步。

好的提问模式：
```
❌ "帮我写个网站"
✅ "我需要一个响应式个人博客，包含首页文章列表、文章详情页、
    暗色模式切换。技术栈用 React + TypeScript + Tailwind CSS，
    不需要后端，文章数据用 JSON 文件管理。"
```

为什么这一步最重要：
- AI 不是读心术，**描述越模糊，产出越偏离预期**
- 约束条件（技术栈、兼容性、性能要求）能帮 AI 做出符合实际的决策
- 清晰的需求 = 更少的返工 = 更高效的合作

记住一个原则：**描述需求花 5 分钟，能省去后面 30 分钟的调试和返工**。

</details>

## 判断题

### Q4：用 AI 做翻译时，指定领域和风格可以显著提升翻译质量。

<details>
<summary>查看答案</summary>

**✅ 正确**

指定领域和风格能显著提升 AI 翻译质量，原因如下：

- **术语准确度**：医学翻译中 "cell" 是"细胞"而非"电池"，法律翻译中 "consideration" 是"对价"而非"考虑"。不指定领域时 AI 可能选错词义。
- **语气一致性**：「正式商务风格」vs「轻松口语化」的翻译差异巨大。明确风格可以确保整篇译文语气统一。
- **实用示例**：
  ```
  ❌ "把这段翻译成英文"
  ✅ "把这段翻译成英文，领域是计算机科学论文，
      风格为学术正式，保留专业术语不翻译"
  ```

实践表明，加上领域和风格约束后，AI 翻译的专业度可提升 30-50%。

</details>

### Q5：AI 数据整理中，告诉 AI 数据的含义比直接让 AI 分析更重要。

<details>
<summary>查看答案</summary>

**✅ 正确**

**告知数据含义是整个 AI 数据分析流程的基石**。

数据本身只是一堆数字和文字——AI 不知道"这个销售额的单位是万元还是元"，也不知道"这个字段代表客户等级"。

关键步骤：
1. **先描述数据**：每列代表什么？单位是什么？枚举值含义是什么？
2. **再定义需求**：想分析什么维度？输出形式是什么？

对比：
```
❌ "帮我分析这个表格" → AI 可能误解数据，得出错误结论
✅ "这是一个电商销售表，'GMV' 是销售额（单位万元），
    'CAT' 是商品类目编码，A=服饰 B=电子 C=食品。
    请分析各类目的 GMV 占比趋势" → 准确的分析结果
```

**数据理解 > 算法选型**。即便是最强模型，在不理解数据含义的情况下也只会"一本正经地胡说八道"。

</details>
