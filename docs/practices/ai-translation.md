# 用 AI 做翻译

翻译不再是逐词查词典——用 AI，你可以轻松完成多语言、专业领域的高质量翻译。本篇教你从"随便翻翻"到"专业翻译"的完整流程。

## 准备工作

- 一个可用的 AI 对话工具（ChatGPT、Claude、DeepSeek 均可）
- 一篇需要翻译的文本（可以是文章、邮件、产品介绍等）

---

## 第 1 步：基础翻译

最直接的用法：把需要翻译的内容贴给 AI，让它直接翻。

### 示例 Prompt

```text
请把下面这段中文翻译成英文：

"近年来，人工智能技术发展迅猛，已经渗透到医疗、教育、金融等各个行业。
它不仅改变了我们的工作方式，也在重塑整个社会的运行模式。"

要求：
- 翻译准确，不要遗漏信息
- 英文表达自然流畅
```

AI 会给出流畅的英文译文。如果觉得某个词或句子不够好，直接追问：

```text
"reshaping"这个词感觉不太对，有没有更自然的表达？
```

### 技巧

- 先说源语言和目标语言
- 对译文风格有要求就直接提出来
- 不满意就追问，AI 不会嫌你烦

---

## 第 2 步：指定风格和领域

同样的内容，在不同场景下翻译方式完全不同。告诉 AI 你的使用场景，它会自动调整措辞和语气。

### 示例 Prompt

```text
请把下面这段产品介绍翻译成英文，用于公司官网。
风格要求：专业、简洁、有说服力，适合面向海外客户。

[粘贴产品介绍文本]
```

不同领域的翻译风格对比：

| 领域 | Prompt 关键描述 | 译文特点 |
|------|----------------|---------|
| 商务邮件 | "正式得体、礼貌清晰" | 敬语、完整句式 |
| 技术文档 | "准确优先、术语一致、简洁" | 短句、无歧义 |
| 文学作品 | "保留原文的美感和文学性" | 修辞、韵律 |

---

## 第 3 步：保持术语一致性

长篇翻译最大的挑战是术语统一。比如 "用户" 到底翻译成 `user` 还是 `customer`？先让 AI 学一下你的术语表。

### 示例 Prompt

```text
我正在翻译一份产品文档，以下是术语对照表，请记住并在后续翻译中严格使用：

- 用户 → user（不是 customer）
- 账号 → account
- 登录 → sign in（不是 log in）
- 注册 → sign up
- 密码 → password
- 验证 → verify

请确认你已经记住了这些术语。
```

AI 确认后，再开始逐段翻译，它会严格遵守你的术语表。如果发现术语不一致，随时纠正：

```text
这里用了 "log in"，请改成 "sign in"。
请把之后所有的 "log in" 都替换为 "sign in"。
```

---

## 第 4 步：翻译审校

好翻译是改出来的。让 AI 自己检查自己的翻译，往往能发现很多问题。

### 自我检查 Prompt

```text
请逐条检查上述英文翻译，从以下维度分析：
1. 是否有语法错误？
2. 是否有信息遗漏或增添？
3. 表达是否地道自然？
4. 术语是否与之前保持一致？

对每条问题，请给出修改建议。
```

### 修改 Prompt

```text
根据你上面的检查结果，请重新翻译一遍，把发现的问题全部修正。
```

如果需要更严格的审校，可以让 AI 用"回译法"——把英文再翻回中文，和原文对照：

```text
请把你翻译的英文再翻回中文，让我和原文对照检查有没有信息偏差。
```

---

## 关键技巧总结

1. **给上下文**：告诉 AI 这段翻译用在什么场景（官网 / 邮件 / 论文 / 社交媒体）
2. **定风格**：明确说明语气（正式 / 轻松 / 专业 / 文艺）
3. **术语管理**：先建术语表，让 AI 记住，全文统一
4. **迭代审校**：让 AI 自查 + 修正，不要一次就过
