# Galileo AI

## 一句话定位

**用文字描述即可生成 UI 设计稿**——你说"我想要一个购物 App 的商品详情页"，Galileo AI 就能给你生成完整的高保真 UI 设计，节省大量线框图绘制时间。

## 核心功能

### 🖥️ 文字 → UI 设计稿
这是 Galileo AI 的核心功能。你只需用自然语言描述你要设计的界面，AI 会自动生成对应的 UI 设计稿。描述越详细，生成结果越精准。例如：

```
Design a travel app home screen with a search bar at the top,
popular destinations carousel, and a bottom navigation bar with
Home, Explore, Bookmarks, and Profile tabs.
```

AI 会生成包含搜索栏、目的地轮播、底部导航栏的完整页面设计，配色和排版都已完成。

### 🧩 设计系统生成
Galileo AI 不仅能生成单页设计，还能基于你的品牌描述生成完整的设计系统——包括配色方案、字体层级、组件库（按钮、输入框、卡片等）。这保证了多页面设计的一致性。

### 📤 Figma 导出
生成的 UI 设计可以一键导出为 Figma 文件，开发者可以直接在 Figma 中继续编辑和调整。这打通了"AI 生成 → 人工精修 → 开发交付"的完整工作流。

### 🔍 组件智能识别
当你输入设计描述时，AI 会自动识别你需要的 UI 组件类型——按钮、卡片、导航栏、表单、图表等，并生成对应的标准组件，而非简单的像素堆砌。

### ⚡ 快速迭代
对生成的 UI 不满意？直接输入修改指令，例如"把背景色改成深色主题""把按钮改成圆角样式"，AI 会快速生成新版本。支持多版本对比，方便选择最佳方案。

## 优势

- **生成质量高**：相比其他 AI 设计工具，Galileo AI 生成的 UI 设计稿视觉质量明显更好，配色协调、排版合理
- **直接可用**：生成结果包含完整的交互元素（按钮、输入框、导航栏），不是简单的静态图片，而是具有一定结构和层级的真实设计稿
- **节省大量线框图时间**：传统设计中，从想法到线框图可能需要几小时甚至几天，Galileo AI 只需几秒钟
- **设计系统思维**：不是孤立地生成每个页面，而是基于设计系统来生成，保证整体一致性

## 不足

- **复杂交互不支持**：对于涉及复杂交互动画、微交互（micro-interaction）、条件显示等场景，Galileo AI 无法表述
- **生成的 UI 有时需微调**：AI 生成的设计稿在间距、对齐等细节上偶有瑕疵，通常需要人工微调才能达到生产级别
- **中文支持有限**：对中文界面设计的支持不如英文，中文排版效果有时不够理想
- **不适合完全定制的品牌设计**：如果你的品牌有非常独特的设计语言，AI 生成的方案可能偏通用化

## 适合谁？

- ✅ UI/UX 设计师——快速产出初稿，节省线框图时间，专注于创意和精细调整
- ✅ 产品经理——把产品需求快速转化为可视化设计，方便与设计师和开发团队沟通
- ✅ 创业者和独立开发者——没有设计资源时，自己用 Galileo AI 出 MVP 的 UI 设计
- ✅ 前端开发者——需要快速搭一个界面原型来验证想法
- ⚠️ 需要像素级精细控制的场景建议用 Figma 进行后续调整
- ⚠️ 移动端复杂交互设计建议使用专业原型工具（如 ProtoPie）补充
